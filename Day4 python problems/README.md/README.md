Day 4: Conditional Statements, Loops and Problem Solving

Concepts Learned

Today I learned how to make decisions in Python using if, if-else, and if-elif-else statements. I also practiced writing nested conditions for situations that need multiple layers of checking, and learned how to combine conditions using logical operators like and, or, and not.

On the loops side, I learned how to automate repetitive tasks using for loops and while loops, and how to control loop execution using break and continue. I also practiced writing nested loops for repeated or grid-like patterns. Overall, today was about building a structured approach to solving problems step by step instead of jumping straight into code.

Problems Solved

I worked through several problems involving conditional statements. This included checking whether a number is positive, negative, or zero, checking whether a number is even or odd, building a simple grade calculator based on marks, finding the largest among three numbers, and checking whether a given year is a leap year.

Next, I moved on to loop based problems. I printed numbers from 1 to 100, printed all even numbers from 1 to 100, calculated the sum of numbers from 1 to N, printed the multiplication table of a given number, and counted the number of digits in a number.

After that, I worked on logic building problems, which felt more challenging. These included reversing a number, checking whether a number is a palindrome, generating the Fibonacci sequence, checking whether a number is prime, and finding all prime numbers between 1 and 100.

As a final mini challenge, I built a Number Analysis Tool. This program takes a number as input and checks whether it is even or odd, whether it is prime, how many digits it has, what the number looks like when reversed, and whether it is a palindrome. All of these results are displayed together in a clean and readable format.

Challenges Faced

The palindrome logic took me some time to fully understand. I initially struggled with it until I realized that I needed to compare the original number with its reversed version, and that I had to keep a separate copy of the original number before reversing it, since the reversing process changes the number itself.

The leap year problem was also tricky because it involves combining multiple logical conditions together. I had to slow down and think through each condition separately before combining them with and and or to get the correct result.

Lessons Learned

The biggest lesson from today is that breaking a problem into plain English steps before writing any code makes the actual implementation much faster and reduces the number of bugs I run into. Once I understand the logic in words, translating it into Python becomes a lot easier.

I also realized that loops and conditionals are the backbone of almost every program. Even something as simple as the Number Analysis Tool is really just a combination of several small logic blocks working together.

Lastly, I learned that writing clean and readable code, using clear variable names and comments, makes it much easier to debug my own code later and to explain my thought process to others.
